## Product Assessment

The strongest product surface is the Sportsbook Board: line-first interaction, compact
board payloads, research drawer context, private Research Slip, MLB readiness, and a
separate strict evidence shelf. This is the right shape for a private daily workflow.

The strongest backend surface is local MLB: ESPN schedule/results, Odds API base lines,
event-specific lines, MLB Stats API logs, NWS/venue/park context, market-history exports,
and strict OOF entry-EV artifacts are all wired.

The weak point is not architecture; it is sample maturity and contract completion.
MLB event-specific markets now settle and the dataset builder enumerates actual
team/player participants from `EventOddsQuote`. Props/team-total EV automation still
needs more settled sample before board promotion, but the participant-aware feature and
artifact contract is now wired.

## Codex Guidance

Official Codex docs currently describe repo skills under `.agents/skills`, with
`AGENTS.md` as the repo instruction file. This repo currently has `.github/skills`
and `.codex/skills`, but no `.agents/skills`. `.github/copilot-instructions.md` remains
useful for Copilot/GitHub, but it is not the canonical Codex skill discovery path.

Recommended consolidation: keep `AGENTS.md` as the project rules, move or mirror repo
skills into `.agents/skills`, and avoid maintaining divergent copies in `.github/skills`
and `.codex/skills` unless a specific tool requires them.
